| 問題 | 根本原因 | 處理方式 |

## 5. 初步觀察與分析方向（供正式報告參考）

以下為 feasibility 階段的初步描述性觀察，不構成正式結論，尚待集群分析驗證。

### 5.1 衝擊程度的初步分層

**高衝擊（超額報酬 < -25%）**：MNDY、TEAM、HUBS、ASAN
- 共同特徵：協作型 / 通用工作流 / CRM 工具；產品功能偏向可被 AI agent 直接替代的流程執行與協作協調。

**中衝擊（超額報酬 -15% 至 -25%）**：NOW、WDAY、ZS、DOCU、SNOW、DDOG
- 特徵混雜：既有深度整合的企業核心系統（NOW、WDAY），也有基礎設施類（ZS、SNOW、DDOG）；後者的衝擊可能更多來自整體 SaaS 估值重定價。

**低衝擊（超額報酬 > -15%）**：CRWD、TWLO、NET、DBX
- 初步觀察：CRWD 為網路安全主動防禦（高更換成本）；NET 為底層網路基礎設施；DBX 收入已停止成長但現金流強健（+36.9% FCF margin）、估值已低；TWLO 為通訊 API 平台。

### 5.2 初步特徵方向假設（待集群分析驗證）

| 假設 | 說明 | 反例或注意事項 |
|------|------|-------------|
| 較高 `sm_to_revenue` → 較高衝擊 | 高行銷依賴可能反映客戶黏著度較低 | DDOG 行銷費用率不高但衝擊中等 |
| 較低 `operating_margin` / 虧損 → 較高衝擊 | 估值壓縮空間較大 | DBX 獲利但增長停滯，衝擊反而最低 |
| 較高 `revenue_growth` → 不一定低衝擊 | MNDY、SNOW 成長率高但衝擊大 | 成長不代表防禦性 |
| 較高 `fcf_margin` → 較低衝擊 | 現金流強→估值壓縮空間小 | DBX 例子支持，但待驗證 |

### 5.3 次要事件日（2025-11-24）的意涵

次要事件窗口 30 家公司平均超額報酬約 **-0.4%**，近乎零。這支持「2025-11-24 為技術公開節點，市場並未在此時形成顯著反應」的操作假設，也間接支持以 2026-01-31 為主事件日的研究設計。

## 6. 集群分析 Prototype 設計規格

（供下一步執行參考）

### 設計決策
- **主版本（Option A）**：5 個特徵欄位 × 13 家公司（排除 MNDY）
  - 欄位：`revenue_growth`, `operating_margin`, `rd_to_revenue`, `sm_to_revenue`, `fcf_margin`
  - 13 家均有完整資料
- **次版本（Option B）**：6 個特徵欄位 × 12 家（再排除 WDAY）
  - 加入 `gross_margin`
  - 用於穩健性確認

### 前處理
- 標準化方式：StandardScaler（z-score）
- 異常值：目前無需特別處理（資料已清理）

### 分群方法建議
- K-Means（k=2, 3, 4）：簡單可解釋，作為 baseline
- 評估指標：Silhouette score、群組內超額報酬分布
- 解讀方式：比較各群的市場衝擊分布 + 特徵輪廓（雷達圖或 bar chart）
